# Задание 2. Проверочный лист безопасности для бизнес-систем

Файл: [`checklist_IB.md`](./checklist_IB.md)

## Выбор разделов

В проверочный лист включены все пять разделов шаблона. Приоритеты:

- **Управление доступом** и **Безопасность данных** - высокий приоритет: API партнёров с избыточными правами, клиенты видят данные чужих клиентов, несколько несогласованных точек регистрации, отсутствие Data Governance.
- **Управление инцидентами** - высокий приоритет: ИБ-специалист не получает все изменения от команд, известная проблема с данными клиентов не была устранена системно.
- **Защита инфраструктуры** и **Специфические аспекты** - средний приоритет: гибридная инфраструктура (ЦОД + облако), внешние интеграции с госорганами и партнёрами.

## Ключевые выводы

**Явные "Нет"** - проблемы, подтверждённые описанием системы:

- Принцип минимальных привилегий не соблюдается (API партнёров дают доступ к чужим данным)
- Единый реестр учётных записей и централизованный контроль доступа отсутствуют
- Политика классификации данных не разработана
- Data Governance не реализован; внутренние потоки данных контролируются несистемно
- Контроль целостности данных нарушен (отображение данных чужого клиента)
- Учёт персональных данных не ведётся должным образом (нарушение 152-ФЗ)
- Плана реагирования на инциденты нет; регулярные аудиты безопасности не проводились

**Явные "Да":**

- Межсетевой экран для внешнего периметра
- Резервное копирование всех баз данных

**Большинство пунктов** имеют статус "Неизвестно" и требуют проверки в ходе запланированного аудита.
