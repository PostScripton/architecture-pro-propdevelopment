# Задание 1. Проверочный лист по безопасности данных

Mindmap классифицирует данные PropDevelopment по стандартам ISO/IEC 27001 и 27002, определяет риски для каждой категории и обосновывает их оценку.

## Структура mindmap

**Центр:** Безопасность данных

### Публичные данные
Каталог объектов, цены, планировки, информация о ЖК, новости о ремонтах.

- Искажение данных `[Значительный]` - ложные данные об объектах снижают доверие клиентов и влекут претензии
- Некачественные данные `[Значительный]` - несколько точек ввода без координации создают устаревшие и противоречивые данные
- Обесценивание данных `[Незначительный]` - устаревшие публичные данные снижают ценность для маркетинга, но не несут прямого ущерба

### Внутренние данные
Данные сотрудников, API-контракты с партнёрами, конфигурации систем, внутренняя отчётность.

- Утечка данных `[Значительный]` - API-контракты с чрезмерными правами доступа раскрывают уязвимости партнёрских интеграций
- Потеря данных `[Значительный]` - потеря конфигураций нарушает работу систем и замедляет восстановление
- Искажение данных `[Значительный]` - некорректные конфигурации могут одновременно нарушить бизнес-процессы нескольких команд

### Конфиденциальные данные
ПД клиентов (ФИО, тел., email), ПД собственников, данные бронирований и онлайн-сделок.

- Утечка данных `[Критический]` - нарушение 152-ФЗ и репутационный ущерб; утечка у конкурента подтверждает актуальность угрозы
- Искажение данных `[Критический]` - отображение данных чужого клиента в личном кабинете - зафиксированная активная проблема
- Потеря данных `[Критический]` - потеря данных сделок влечёт юридические и финансовые последствия
- Некачественные данные `[Значительный]` - несколько точек регистрации без координации порождают дублирование данных клиентов

### Секретные данные
Учётные данные БД и API-ключи, финансовые транзакции, юридические документы сделок, данные интеграций с госорганами.

- Утечка данных `[Критический]` - компрометация учётных данных открывает полный доступ ко всем системам компании
- Потеря данных `[Критический]` - потеря транзакционных и юридических данных ведёт к финансовым потерям и судебным искам
- Искажение данных `[Критический]` - подмена финансовых данных сделок создаёт прямые возможности для мошенничества
