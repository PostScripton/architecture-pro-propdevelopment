## Файлы

### C4_context_diagram.drawio.xml

Диаграмма контекста C4 для новых функций умного дома.

Участники:
- **Собственник** (Person) - управляет доступом в дом и на парковку через мобильное приложение.
- **PropDevelopment** (System boundary) - включает мобильное приложение и новый компонент `smart-home-service`.
- **Платформа умного дома (партнер)** (External Software System) - предоставляет API домофона (распознавание лиц) и шлагбаума (распознавание номеров).

Потоки:
- Собственник -> PropDevelopment: управление устройствами через мобильное приложение (REST).
- PropDevelopment -> Партнер: команды управления и списки доступа (REST/HTTPS, OAuth 2.0).
- Партнер -> PropDevelopment: события доступа (проход, открытие) по Webhook с подписью HMAC-SHA256.

Для просмотра диаграммы откройте файл в [draw.io](https://app.diagrams.net/).

### C4_container_model_updated.drawio.xml

Обновлённая диаграмма контейнеров PropDevelopment на основе шаблона `template_C4_container_model.drawio.xml`.

Добавлены элементы:
- **smart-home-service** (Container, Kotlin/SpringBoot) - новый сервис в группе ЖКУ. Адаптер, инкапсулирующий взаимодействие с платформой партнера. Мобильное приложение обращается только к нему, не напрямую к партнеру.
- **Платформа умного дома (партнер)** (External Software System) - вне границы PropDevelopment. Получает запросы на управление устройствами и отправляет события доступа.
- **Firewall** - между PropDevelopment и платформой партнера.
- Три новых связи: мобильное приложение -> smart-home-service [REST], smart-home-service -> партнер [REST/HTTPS, OAuth 2.0], партнер -> smart-home-service [Webhook].

Оранжевой обводкой на диаграмме выделены новые элементы и связи, добавленные для интеграции сервисов умного дома. Остальной ландшафт соответствует исходному шаблону.

### requirements.md

Список требований к внешним интеграциям, структурированный по трём блокам:

1. **Требования к безопасности** - шифрование TLS 1.3, защита биометрии, изоляция данных по ЖК, минимизация данных, журналирование событий доступа (90 дней), rate limiting, соответствие ФЗ-152.

2. **Протоколы аутентификации и авторизации** - OIDC/OAuth 2.0 для собственников через Keycloak, JWT с claims (роль, ЖК), RBAC для прав на устройства, OAuth 2.0 Client Credentials для сервис-к-сервису с партнером, HMAC-SHA256 для верификации Webhook.

3. **Организация взаимодействия** - паттерн Adapter (smart-home-service), синхронные команды (REST/HTTPS, таймаут 5 с), асинхронные события (Webhook), версионирование API-контракта, circuit breaker для изоляции отказов партнера.
